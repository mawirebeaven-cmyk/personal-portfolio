# Personal Portfolio Website

A modern, responsive personal portfolio website built with HTML, CSS, and JavaScript.

## 📁 Project Structure

```
├── index.html           # Home page
├── about.html           # About me page
├── projects.html        # Projects showcase
├── blog.html            # Blog/Articles
├── contact.html         # Contact form
├── calculator.html      # Calculator tool
├── todo.html            # Todo list app
├── style.css            # Main stylesheet
├── script.js            # JavaScript functionality
├── images/              # Image assets
├── vercel.json          # Vercel deployment config
└── README.md            # This file
```

## 🌟 Features

- **Home Page** - Introduction and hero section
- **About** - Personal background and skills
- **Projects** - Showcase of your work and projects
- **Blog** - Articles and blog posts
- **Contact** - Contact form to reach out
- **Tools** - Includes a calculator and todo list
- **Responsive Design** - Works on desktop, tablet, and mobile
- **Static Site** - No backend required, easy to deploy

## 🚀 Deployment

This site is deployed on Vercel. To deploy:

1. Push your changes to Git
2. Connect your repository to Vercel
3. Vercel will automatically build and deploy

The `vercel.json` configuration ensures all static files are properly served.

## 📝 Pages

- **index.html** - Main landing page
- **about.html** - About section with background info
- **projects.html** - Portfolio of your projects
- **blog.html** - Blog or articles section
- **contact.html** - Contact form
- **calculator.html** - Interactive calculator
- **todo.html** - Todo list application

## 🎨 Styling

The website uses centralized styling through `style.css`. Customize colors, fonts, and layouts in this file.

## 💻 JavaScript

Interactive features and functionality are handled in `script.js`.

## 📸 Images

Store all images in the `images/` folder for organized asset management.

## 🔧 Development

To work locally:

1. Clone the repository
2. Open `index.html` in your browser
3. Edit HTML, CSS, or JavaScript files as needed
4. Test changes locally before deploying
